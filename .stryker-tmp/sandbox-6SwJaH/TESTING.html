Quick Start

This guide will help you test the Online Picket Line browser extension.

1. Installation

Chrome/Edge:

  1. Open chrome://extensions or edge://extensions
  2. Enable "Developer mode"
  3. Click "Load unpacked"
  4. Select the extension directory

Firefox:

  1. First, copy manifest-v2 to manifest: 
    cp manifest-v2.json manifest.json
  2. Open about:debugging#/runtime/this-firefox
  3. Click "Load Temporary Add-on"
  4. Select any file in the extension directory
Note: For Firefox testing, you'll need to temporarily replace the manifest.json file with manifest-v2.json, as Firefox currently uses Manifest V2.

2. Configure Test Data

Since the actual API may not be available, you can test with mock data by:

Option A: Modify the API URL

Edit api-service.js and change the API URL to a test endpoint or mock server:

const API_BASE_URL = 'http://localhost:3000';

Option B: Use Sample Data

The extension includes sample-data.json with example labor actions. Set up a local server:

python -m http.server 3000

Or use any static file server to serve the sample data.

3. Testing Scenarios

Test 1: Banner Mode (Default)

  1. Click the extension icon
  2. Ensure "Show Banner" is selected
  3. Visit a site in your test data (e.g., example.com if using sample data)
  4. You should see a red banner at the bottom of the page
  5. Verify the banner shows labor action details
  6. Test the close button

Test 2: Block Mode

  1. Click the extension icon
  2. Select "Block Access"
  3. Visit a site in your test data
  4. You should be redirected to a block page
  5. Verify the block page shows labor action details
  6. Test "Proceed Anyway" button
  7. Test "Go Back" button

Test 3: Settings and Stats

  1. Click the extension icon to open the popup
  2. Verify the stats show the number of active labor actions
  3. Click "Refresh Labor Actions"
  4. Verify the timestamp updates
  5. Switch between Banner and Block modes
  6. Verify settings are saved

Test 4: URL Matching

The extension should match URLs in several ways:

  • Exact domain: example.com matches example.com
  • Subdomain: www.example.com matches example.com
  • Company name: If configured, matches based on company name

5. Debugging Tips

Check Console Logs:

  • Extension background: chrome://extensions ‚Üí Details ‚Üí Inspect views: service worker
  • Content script: F12 on any webpage ‚Üí Console tab
  • Popup: Right-click extension icon ‚Üí Inspect popup

Common Issues:

  • No banner/block: Check if URL matches test data
  • API errors: Verify the API endpoint is accessible
  • Permissions: Ensure all permissions are granted
  • Cache: Clear extension storage if data seems stale

Need Help?

If you encounter issues:

  1. Check the browser console for errors
  2. Verify the API endpoint is accessible
  3. Ensure all files are present in the extension directory
  4. Try reloading the extension
  5. Clear extension storage and try again
